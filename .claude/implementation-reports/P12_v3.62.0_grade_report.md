- **Phase:** P12 (70.md)

- **Feature flag:** `gradeCertainty` — **OFF by default**. When OFF, the existing single-outcome GRADE tab renders **byte-identical** to before (the legacy body was preserved verbatim). When ON, the per-outcome, audited, lockable workspace replaces it.
- **User-facing wording:** GRADE terminology throughout; no "AI" wording.

## What was implemented

An upgrade of PecanRev's existing (single-outcome, blob-based) GRADE feature into a **per-outcome** certainty-of-evidence workspace that **prefills domain suggestions from data the app already computes** (RoB summary, I²/heterogeneity, effect estimate + CI width, Egger), **requires human confirmation** for the final certainty (High/Moderate/Low/Very low), records an **audit trail**, supports **locking** a finalized judgment, and generates a **Summary-of-Findings table** (with footnotes) that also populates the previously-dangling manuscript SoF certainty column. Domain keys are kept **engine-agnostic (CINeMA-ready)** so NMA certainty can feed the same model later.

## Security / privacy

- All computation is deterministic and server-side; prefill only reads the project's own data. Suggestions are `source:'auto'` and **never become the final judgment without an explicit human save** (a locked outcome rejects writes with 409). Every change (save/lock/unlock) is audited (before/after + author). Reads gate on project view; writes/lock on edit (owner/leader) — server-side.

## Performance

- Per-outcome meta is computed on demand from the studies blob (bounded by outcome count); no persisted heavy computation, no server-wide lag.

## Known limitations & concrete solutions

1. **Per-outcome RoB association is best-effort.** RoB assessments aren't hard-linked to outcome pair keys, so the RoB→GRADE mapping matches by `outcomeId`/label with a project-level fallback.
   *Solution:* when the extraction/RoB model gains a first-class outcome id (P13/P14 area), key RoB assessments to the same `pair.key` and drop the fallback.
2. **Publication-bias suggestion intentionally differs from the legacy heuristic.** The new engine does NOT downgrade on `k<10` alone (GRADE cautions against that — few studies means you can't *assess* bias, not that it exists); only a significant Egger (p<0.05, k≥3) suggests downgrading. The legacy tab (flag off) keeps the old behavior.
   *Solution:* documented as a deliberate methodological correction; both are reviewer-overridable. If a stricter house policy is wanted, expose the threshold in a future Ops GRADE-policy block.
3. **CINeMA (NMA certainty) is architecture-only.** Domain keys + the NMA-warning→domain mapping exist, but per-comparison NMA certainty isn't computed yet.
   *Solution:* a follow-up can feed `runNetworkMetaAnalysis().warnings` through `mapNmaWarningToDomain` into `GradeOutcomeAssessment` rows keyed per comparison — the model already supports it.
