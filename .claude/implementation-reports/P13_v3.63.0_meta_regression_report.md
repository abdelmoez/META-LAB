## What was implemented

Random-effects meta-regression (continuous + categorical covariates) as a natural extension of the existing Subgroup Analysis, with a weighted **bubble plot** (fitted line + 95% CI band), full result reporting, statistical guardrails, and exports. Deterministic engine, **validated to 1e-9** against hand-computed and independently-recomputed references.

## Files changed

- NEW `src/research-engine/statistics/metaRegression.js` — `metaRegression(studies, opts)`. Random-effects via **method of moments (multivariable DerSimonian–Laird residual)** and **REML** (Fisher scoring, MM-initialised, MM fallback). Reuses `nma/linalg.js` (Cholesky inverse/solve) for (X'WX)⁻¹ and `math-helpers.js` (normalCDF, chiSquareCDF, Z975). ## Result shape (engine)

`{ ok, k, kDropped, model:'random', method, intercept:{coef,se,z,pval,ciLo,ciHi}, moderators:[{name,level?,coef,se,z,pval,ciLo,ciHi,kind,reference?}], tau2, tau2Before, tau2Reduction, residual:{QE,QEp,df}, R2, I2resid, omnibus:{QM,df,pval}, warnings:[{type,message,n?}], bubble:{points:[{x,y,weight,label,studyId}], line, band}, provenance }`. Bubble data (x=covariate, y=effect, radius∝weight, fitted line + 95% band) is engine output → one source of truth for UI + export.

## Security / privacy

Deterministic pure math; server route is stateless and flag-gated. No external calls, no persistence, no PII. Reads only the project's own study data.

## Performance

Client-side `useMemo` for interactivity (like Subgroup); the server route is a thin stateless wrapper. Matrix ops are O(k·p²) — trivial for realistic k. No server-wide lag, no jobs needed.

## Statistical-integrity considerations

Every result carries honest guardrails: `TOO_FEW_STUDIES` (k<3 or k<nMod+2 → ok:false; k<10·covariates → warn), `TOO_MANY_COVARIATES`, `MISSING_COVARIATE_DATA`, `MULTIPLE_TESTING`, `ECOLOGICAL_BIAS` (always — aggregate covariate ≠ patient-level), `OBSERVATIONAL_COVARIATE` (always — associations, not causation), `REML_FALLBACK`. The UI narrative/methods/manuscript text is association-only and never implies causality.

## Known limitations & concrete solutions

1. **UI exposes univariate meta-regression; the engine already supports multivariable** (`opts.covariates`).
   *Solution:* the engine + design matrix are multivariable-ready; a follow-up adds a multi-covariate picker to the UI (the results table already renders N moderators).
2. **Covariates come from existing study fields (auto-detected) + type override; no dedicated CSV-covariate import or manual per-study covariate editor yet.**
   *Solution:* study fields already cover year/n/followup/dose/region/design; a follow-up can add a CSV-column import that writes covariate values onto the studies blob (the engine consumes any field by name).
3. **Bubble CI band is drawn for a single continuous covariate; categorical falls back to a grouped scatter + the table.**
   *Solution:* documented in the UI; a partial-residual (added-variable) bubble plot for multivariable models is the next visualization step.
