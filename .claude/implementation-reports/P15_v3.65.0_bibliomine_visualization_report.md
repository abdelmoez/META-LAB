## What was implemented

Upload seed-review PDFs → extract their reference list → resolve references (CrossRef/PubMed/OpenAlex) → dedupe + import into screening with seed provenance → OpenAlex backward/forward **citation chasing** (queued, depth/limit-capped, cancellable) → **choropleth map** of study geography + **characteristic histograms** (study type, sample size, year, region, design, RoB).

## Security / privacy

Deterministic parsing + dedup are server-side/pure. Client extracts PDF text locally and posts only text — the raw PDF never leaves the browser. External provider calls go through the hardened rate-limited/backed-off connectors, are **double-gated** (`citationMining` flag + `CITATION_MINING_LIVE_RESOLVE` env), and the chase is bounded (depth≤3, ≤2000 candidates, per-node cap, cycle-safe, cancellable). Only bibliographic metadata / legal OA — no paywall bypass. Import is permission-gated (leader/owner) and preserves provenance + PRISMA accounting via the shared dedup import path.

## Adversarial verification pass (4 independent refutation reviewers)

Before committing, an adversarial workflow tried to *refute* the P15 safety claims. Verdicts: **bounded-chase-external-fanout HOLDS** (depth/candidate caps clamped at enqueue AND execution, cycle-safe job-scoped dedup, cancel re-read per node, live calls off by default), **flag-off inertness HOLDS** (no nav entry / no calls when off; a forged `?tab=citation` deep-link only renders a harmless disabled-note shell, mirroring NMA/Living Review), **no paywall bypass / no user-facing "AI" / parser-robustness HOLDS**. The **import-integrity** lens was **VIOLATED**; all findings fixed pre-commit:

1. **[HIGH] Import bypassed the shared gating** — `importCandidates` called `dedupeAndInsertRecords` without a `maxRecords`, so it used the 100k code default instead of the resolved per-project cap, and skipped the admin import kill-switch + product-tier entitlement/limit that file/Pecan imports enforce. **Fixed:** `importCandidates` now runs the SAME gating as `screeningController.importRecords` — `getMetaSiftSettings().allowImport` kill-switch, `requireEntitlement('screening.import')` + `requireLimit(owner,'screening.maxRecordsPerProject')`, and `resolveScreeningUploadLimit({settings}) → maxRecords` (controller maps `TierLimitError`/`IMPORT_DISABLED`/`CAPACITY`).
2. **[MED] Provenance silently truncated** — per-record provenance was the last key in `rawData`, after a 6k-char abstract, so the 2k `rawData` cap dropped it. **Fixed:** `citationProvenance` is now the FIRST key (survives truncation; abstract also lands in its own column).
3. **[LOW] `imported` flag over-set** — every processed candidate (incl. dedup-rejected identity-less rows) was marked `imported:true`. **Fixed:** only candidates eligible to land (have title/DOI/PMID) are marked, so identity-less rows stay re-importable.
4. **[LOW] Seed/candidate dedup key mismatch** — seeds stored a `'seed:'`-namespaced key while produced candidates checked the bare key, so a work that was both could be recorded twice. **Fixed:** seeds register both keys.

Re-verified after fixes: service 6/6 + integration 15/15 + `screening-api` 20/20 regression green. Two accepted LOW notes (documented): cancellation is observed at node boundaries (one node's bounded work completes first — bounded latency, not a fan-out breach); the forged-deep-link disabled shell is the intended fail-safe.

## Known limitations & concrete solutions

1. **Resolved seed *references* aren't directly importable** — only citation-chase *candidates* are, so importing a seed review's own references currently requires chasing from them. *(Being addressed in the recs round: `resolveSeed` will also create `CitationCandidate` rows (source `reference`) so seed references import via the same path.)*
2. **Server-side PDF text extraction is client-assisted** — there is no server PDF library, so reference text is extracted in the browser (pdf.js) and posted. *Solution:* acceptable + private (raw PDF stays local); a server-side extractor could be added for headless/bulk ingestion later.
3. **Dashboard cards (studyMap / characteristicsHistogram) not added to the P8 composer** — the public payload is server-built + sanitized, so new cards need the server to embed sanitized geography/characteristics. *Solution:* a focused follow-up adds the card types + server payload builder + renderer branch.
4. **Reference-parser accuracy caveats** (documented in-code): a title containing `". "` can truncate; journal abbreviations kept verbatim; strict country normalization sends ambiguous names to the unmapped bucket rather than guessing. *Solution:* resolution against CrossRef/OpenAlex corrects parsed fields when identifiers resolve.
