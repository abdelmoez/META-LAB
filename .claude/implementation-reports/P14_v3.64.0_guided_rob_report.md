# P14 — Guided Risk-of-Bias Appraisal (RoB 2 + ROBINS-I) — Implementation Report

- **Phase:** P14 (70.md)
- **Version:** v3.64.0 (continuing the repo's 3.x semver per 70.md's escape clause; P14→v0.14.0 maps here to v3.64.0).
- **Feature flag:** `guidedRobAppraisal` — **OFF by default**, functionally requires `rob_engine_v2`. With it OFF the RoB workspace is unchanged (RoB 2 only; no instrument selector, appraisal, or validation; no non-RoB2 assessments creatable). RoB 2 behavior is **byte-identical** to before.
- **User-facing wording:** "Guided appraisal" / "Suggested" / "Appraisal". No user-facing "AI" wording (verified by an adversarial reviewer — the only `ai*` tokens are non-rendered JSON API field names).

## What was implemented

Upgraded the existing manual RoB workspace so it can **read a study's abstract/full text and SUGGEST per-domain signalling answers** with a quoted supporting sentence, source location, and confidence, then derive a proposed domain judgment — always as a reviewer **suggestion that never overwrites a human judgment**. Adds **ROBINS-I** (non-randomized studies) alongside RoB 2, a machine-vs-human **agreement validation** (weighted κ) view, and 5-level traffic-light visuals.

## The never-overwrite-human seam (safety-critical)

Machine output writes ONLY `RobDomainJudgment.proposedJudgment` + answer-level fields (`response/evidenceQuote/evidenceLocator/aiSuggested/aiConfidence`). Human decisions (`finalJudgment`/`overridden`/`overrideJustification`) are never touched by the appraise path. Appraise skips any question a human already answered (unless an explicit, audited `force`); a human edit demotes any prior suggestion to a protected human row. Every appraisal is audited (`ROB_APPRAISE` with model/version/coverage).

## Adversarial verification pass (5 independent refutation reviewers)

Before committing, an adversarial workflow tried to *refute* the invariants. Verdicts: **never-overwrite-human HOLDS**, **RoB 2 byte-identical HOLDS**, **ROBINS-I methodology HOLDS**, no user-facing "AI" **HOLDS**. It found **4 real defects, all fixed pre-commit**:

1. **[HIGH] Weighted-κ category order** — ROBINS-I validation ordered categories by display order, putting the non-ordinal `ni` at the top of the severity scale and inverting the disagreement weights. **Fixed:** ROBINSI now exposes `judgmentOrder` (severity-ascending, `ni` between Moderate/Serious); the validation uses it (RoB 2's display order already equals its severity order).
2. **[MED] Flag gating gap** — `createAssessment` gated instrument choice only on `rob_engine_v2`, so a ROBINS-I assessment could be created while `guidedRobAppraisal` was OFF. **Fixed:** non-RoB2 instruments now require `guidedRobAppraisal` (400 otherwise) → RoB2-only when off.
3. **[MED] Validation κ inflation** — pairing machine-proposed vs `finalJudgment` counted `finalise`-auto-copied agreements (final≡proposed) as real agreement. **Fixed:** validation now pairs only domains the reviewer **explicitly** judged (`overridden === true`), so agreement isn't manufactured; `n` is surfaced.
4. **[MED] Redundant network call** — `guidedRobAppraisalEnabled()` fetched `/api/settings/public` separately on mount. **Fixed:** both RoB flag helpers now share one briefly-cached settings fetch → no extra round-trip vs pre-P14.

## Security / privacy

Deterministic, server-side appraisal (full text supplied by the client from the already-authorized PDF; no server PDF fetch). Suggestions are provenance-stamped and reviewable/reversible; human sign-off required; permission checks server-side.

## Known limitations & concrete solutions

1. **[LOW] ROBINS-I Q1.6 reachable on the follow-up-time-splitting path** (found adversarially) — the published tool omits Q1.6 there, so a correct time-varying (g-methods) analysis could be spuriously downgraded to Serious/Critical.
   *Solution:* gate Q1.6's `branch` to exclude the time-splitting path (add the split indicator to its `allOf`); low severity/edge-case, documented for a focused follow-up.
2. **[LOW] `force` overwrites another reviewer's *uncommitted* signalling answers** (by design, audited) — it never touches a committed domain/overall judgment, so the safety invariant holds, but it is a residual answer-overwrite behind an explicit operator flag.
   *Solution:* scope `force` to the requesting reviewer's own answers, or require a second confirmation; documented.
3. **Validation reflects domains with an explicit human judgment** (by the FIX-3 design) — early in a project with few overrides, `n` is small.
   *Solution:* already surfaced (`n` + disagreement queue); a future "reviewer confirmed" flag distinct from finalise-auto-copy would let accepted-suggestion agreements count without inflation.
