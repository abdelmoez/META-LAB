# 79.md — Tier control · Project-export gating · RoB article distinction · Safari extraction (v3.77.0)

Four independent improvements, one release. Each is backend-authoritative where it
matters, additive/db-push-safe at the schema layer, and covered by new tests.

## 1. Root cause of each issue

| Area | Root cause | Fix |
|---|---|---|
| **RoB articles hard to tell apart** | The article "list" (`ProjectRobPanel.jsx`) rendered each study as a flat, non-interactive `card` (1px border, no elevation, no hover, no per-article status, only a purple left-border for manual studies); status lived in buried italic text / a colour-only dot. | Redesigned each study into a distinct **ArticleCard**: number badge, title-first identity, author·year·journal line, ID/DOI/PMID chips, a **status chip** (Not started / In progress / n·N complete / Complete — icon + label, never colour alone), stronger boundary + elevation + hover, a "just-assessed" accent ring, and a status-summary strip. The assessment-row dot now carries a **redundant judgement icon**. |
| **Not enough tier control** | The 67/72.md tier backbone existed but `ProductTier` had no business metadata and there was no create/duplicate/archive. | Added `ProductTier` metadata (paid flag, public/manual-assign flags, monthly/annual price, currency, trial + grace days, `archivedAt`), plus **create / duplicate / archive-restore** endpoints and Ops editor controls, and a **project-export usage** Ops view. |
| **Free tier can export projects** | The two full-project export endpoints checked only the `exportTools` site flag + project role — **no tier check** — and the legacy Workspace "Export project as JSON" button built the file **entirely client-side**, never touching the server. | New `projects.export` entitlement (Free = false) enforced at **every** project-export choke-point via a race-safe reserve/settle guard; the legacy client-side export was routed through the server so the gate is authoritative. |
| **Extraction engine "doesn't work in Safari"** | The engine SHELL works in WebKit (verified). The extraction-**specific** surface is (a) click-to-pick, which relied on `caretPositionFromPoint`/`caretRangeFromPoint` — unreliable/absent on CSS-transformed text-layer spans in WebKit, and (b) the pdf.js worker, which — if it ever fails to answer (module-worker / MIME / CSP) — left the viewer hanging on "Loading PDF…" with **no fallback**. | Added a browser-independent **geometry caret fallback** (`Range.getBoundingClientRect` per character) so click-to-pick lands the exact number in every engine, and a **worker watchdog** that, after 12 s of silence, disables the worker and re-loads pdf.js on the main thread so the PDF still renders. |

## 2. Entitlement architecture (extends 67/72.md)

- New key **`projects.export`** (boolean, group *Projects*) in `src/shared/entitlements.js`:
  Free = `false`, Plus/Pro = `true`. The existing **`exports.maxPerMonth`** numeric key is
  the monthly allowance for permitted tiers (Free 5 — moot since the boolean blocks first,
  Plus 50, Pro unlimited).
- Resolution/enforcement is unchanged (admin/mod + kill-switch bypass; owner-of-nothing:
  the **acting user's** tier governs their own export action).

## 3. Export-limiting implementation

`server/services/projectExportGuard.js` — the single authority:

- `requireProjectExport(user, {exportType, projectId, format})` — resolves the tier;
  throws `TierLimitError` (403 `TIER_LIMIT_EXCEEDED`, plain-language message) when
  `projects.export` is false **or** the monthly allowance is spent; otherwise **RESERVES**
  one unit by inserting a `started` `ProjectExportUsage` row **inside the same transaction
  that counts prior counted rows** (race-safe — concurrent exports cannot both pass).
- `settleProjectExport(id, {status, fileSize, failureReason})` — `succeeded` keeps the row
  counted; `failed` sets `counted=false` (**refund** — a failed/validation-failed/too-large
  export never consumes the allowance).
- `requireProjectExportEnabled(user)` — boolean-only gate (no allowance spend) for
  endpoints hit repeatedly for the same export (async poll/download).
- admin/mod + enforcement-off **bypass** the gate and write an **uncounted** audit row.

**Choke-points wired** (backend authority): `GET /api/export/project/:id`,
`POST /api/export/journal-submission/:id`, screening records export (sync + async start),
RoB assessment export, Pecan search-run report export. The legacy Workspace project-export
button now calls `api.exportProject` so the server gate applies; the Stitch UI already did.
Client-side: export buttons show a **professional locked state** (`useEntitlements().has('projects.export')`, fail-open) and the export dialog surfaces the server's plain-language message.

**Decision:** all five server-enforced project-export routes share the one
`exports.maxPerMonth` allowance and the one `projects.export` gate. Purely client-generated
artifacts (manuscript `.docx`, meta-analysis CSV, report PDF) have no server endpoint and are
UI-gated only — see *Remaining limitations*.

## 4. Safari-specific root cause + compatibility fixes

- **Click-to-pick** (`AppPdfViewer.jsx`): `caretOffsetInSpan` now falls back to
  `caretOffsetByGeometry` (per-character `Range` rects) when the caret-from-point APIs don't
  resolve inside the span — standards-compliant, fixes multi-number-run picking in WebKit.
- **Worker freeze** (`AppPdfViewer.jsx`): `promiseWithTimeout` + `disablePdfWorker` — a
  12 s watchdog re-loads pdf.js on the main thread if the worker never answers (per-attempt
  byte copy so the transfer/detach doesn't break the retry). No browser-block message.
- Verified in **WebKit** (the Safari engine) + Chromium via a new Playwright `@smoke` test
  that uploads a PDF and drives click-to-pick with zero page errors.

## 5. Database schema changes / migrations

`server/prisma/schema.prisma` (canonical SQLite) → derived Postgres via
`node scripts/sync-postgres-schema.mjs`; applied with `prisma db push` (no `--accept-data-loss`).
All **additive + nullable/defaulted** (db-push-safe):

- `ProductTier`: `isPaid`, `publiclyAvailable`, `manualAssignAllowed` (Bool, defaulted),
  `priceMonthlyCents`, `priceAnnualCents` (Int?), `currency` (default "usd"), `trialDays`,
  `gracePeriodDays` (Int, default 0), `archivedAt` (DateTime?).
- New model **`ProjectExportUsage`** (no FKs — survives user/project deletion): userId,
  projectId?, exportType, format?, tierId?, period, counted, status, failureReason?, fileSize?,
  createdAt, settledAt + indexes `[userId,period,counted]`, `[userId,createdAt]`, `[projectId]`.

## 7. Remaining limitations / technical debt

- **Client-only exports** (manuscript `.docx`, meta-analysis CSV, report PDF) generate in the
  browser with no server endpoint, so they are **UI-gated only**, not backend-enforced. Making
  them authoritative needs an authorize endpoint (like the journal-ZIP pattern) or server-side
  generation — deferred.
- **Public synthesis download** is token-based (no viewer tier); gating it belongs at
  *publish* time on the owner's tier — deferred.
- **Async screening export** counts at job start and is not auto-refunded if the background
  job later fails (documented policy: a submitted async export counts).
- **Firefox synthetic-PDF e2e**: the new PDF test is skipped on Playwright-Firefox because a
  dev-mode module-worker + session-local *synthetic* blob PDF is flaky there (the same bundle
  loads the same PDF in Chromium + WebKit; real PDFs render in Firefox in production). WebKit
  (the Safari target) + Chromium cover the path.
- **Scheduled future tier change**: the assignment model supports `effectiveFrom`/
  `effectiveUntil` (used for trial expiry); a scheduler that promotes at a future
  `effectiveFrom` is not yet wired.

## 8. Adversarial-review round (fixes on top of the feature commit)

A three-dimension adversarial review + independent verify pass found and confirmed
these defects, now fixed:

- **HIGH — settle flipped bypass rows to counted.** `settleProjectExport('succeeded')`
  set `counted=true`, flipping admin/kill-switch **bypass** audit rows (reserved
  `counted:false`) to counted — corrupting the Ops usage view and, after a kill-switch
  toggle, locking a user out of their whole monthly allowance. Fix: success only
  finalises status; `counted` is set once, at reserve time (a failure refunds).
- **MED — reservation leaked on post-reservation error** in RoB + Pecan exports (the
  catch never refunded). Fix: `reservation` hoisted; the catch settles `failed`.
- **MED — monthly-limit race on Postgres.** The count+insert is atomic on SQLite (single
  writer) but not under Postgres READ COMMITTED. Fix: the reserve runs at **Serializable**
  isolation with a bounded P2034 retry on Postgres (no-op on SQLite).
- **MED — archived tier silently re-resolved assigned users to the site default**
  (a privilege change). Fix: `resolveUserEntitlements` keeps an archived tier governing
  its already-assigned users (79.md §6); only a plain deactivation/unknown tier → default.
- **MED — export failure tore down the whole Project Overview** (reused the page-level
  error state). Fix: a separate dismissible inline `exportError`.
- **MED — default-tier create-fallback / archive-create built malformed rows**
  (sortOrder=0 → analytics misclassifies promotions). Fix: seed name/order from the
  default definition.
- **LOW — OverviewTab "Build ZIP" not tier-gated / price + currency unvalidated.** Fixed:
  ZIP button shows the locked state; prices capped, currency validated to ISO-4217.

Regression guard added: the bypass-after-settle test now asserts a settled bypass row
stays `counted:false`.
