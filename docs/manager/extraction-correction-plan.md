## 1. Current-state file inventory (§7.1)

Derived from a 12-agent parallel audit of the repository at v3.71.2 plus direct reads. One paragraph per file/group; disposition = keep / extend (additive) / rewrite-internal / new.

### 1.1 Grid engine — `src/research-engine/extraction/pdfTextGrid.js` (623 lines) — EXTEND
Exports `normalizeItems`, `itemsToRows`, `detectColumns`, `buildGrid`, `gridFromRegion`, `looksNumericColumn`, `looksCiColumn`, `detectHeaderSpan`, `partitionRows`, `mergeContinuationColumns`, `NUMERIC_CELL_PATTERNS`, `CI_CELL_PATTERNS`. Pure, deterministic, golden-tested (`tests/unit/extraction/pdfTextGrid.test.js`). Input: raw pdf.js items `{str, transform, width, height}` (y grows UP; baseline origin). Output today: `{cells: string[][], boxes}` matrices. Deficiencies vs 4.md §12: no token repair (mid-word splits like `u nivariate` survive), no caption capture (footnote leaders exist via `partitionRows` but caption text leaks into cells), column inference clusters start-x only (left-anchored — right-aligned/decimal-aligned numeric columns mis-split), no colSpan/headerPath, no wrapped-row merging or row hierarchy (`indentLevel`/`parentRow`), no structural confidence/warnings, no §10.2 grid contract. Disposition: keep every existing export byte-compatible (they are load-bearing for TableRegionMapper, tableShape, and golden tests); add the staged pipeline as new exports composing into a new top-level `buildTableGrid()` (§4 contract C2 below), then migrate consumers.

### 1.2 Token snapper — `src/research-engine/extraction/numberTokens.js` (280 lines) — EXTEND (behind cellGrammar)
Exports `snapNumberToken(runStr, offset)`, `findNumberTokens`, `parseNumberList`; kinds `number/percent/range/ratioCI/pair/meanSd` with richest-match overlap resolution; deterministic, never throws; well tested. Deficiencies vs §13: no `P` kind (clicking `<0.001` loses the operator), no `MISSING` kind, no count+percent composite (`12 (9.6%)` parses as two unrelated tokens), no `64 of 125` word form, no bracket/comma CI forms (`(1.40, 3.57)`), no leading-dot decimals (`.56`), descending CI silently discarded rather than flagged, offset-only (no selectionRange), strict containment (no nearest-token snap). Disposition: becomes the internal engine of the new shared `cellGrammar.js`; missing forms added here; all six divergent parsers in the repo (below) route through cellGrammar.

### 1.3 Fragmented number parsers — CONSOLIDATE into cellGrammar
Six independent numeric grammars exist with contradictory behavior: `numberTokens.js`; `TableRegionMapper.jsx:72-96` `parseNumCell`+`splitCI` (silently REORDERS reversed CI bounds via Math.min/max — violates §13.1); `patternExtract.js` `toNum`/`match*` (accepts `.04`, has p-value operator preservation the others lack); `tableParse.js` `looksNumeric`; `pdfTextGrid.js` cell patterns (column-typing only — acceptable to keep for typing); `AssistedExtractionPanel.jsx:44` `firstNumber` (mandated deleted by §13.3/§21.2). Disposition: new `cellGrammar.js` is the single parser for cell values and click snapping; `parseNumCell`/`splitCI`/`firstNumber` are migrated and deleted; `patternExtract` keeps its sentence-level statistics engine but its p-value/number primitives are aligned in a follow-up test (its behavior is already golden-tested and prose-scoped).

### 1.4 Table shape — `src/research-engine/extraction/tableShape.js` (193 lines) — REWRITE-INTERNAL (API extended)
`detectTableShape({cells,boxes})` → `{shape: 'direct-effect'|'dichotomous'|'continuous'|'unknown', columnTags, rowKind, headerRows, confidence}`. Header-vocabulary + body-pattern scoring; runner-up score computed then DISCARDED (no alternates); no `pico` parameter; no arms-in-columns or single-arm shapes; MD/SMD/RD effect headers missing from EFFECT_RE. Disposition: extend to the 4.md 5-shape vocabulary (renames: direct-effect→`effect-per-row`, dichotomous→`two-by-two`, continuous→`mean-sd`; new: `arms-in-columns`, `single-arm`), add `evidence[]` + `alternates[]` (≤0.15 rule), accept `(grid, pico)` and delegate arm matching to new `armMatch.js`. All call sites + tests migrated in the same increment (§9.3 — no dead vocabulary left behind).

### 1.5 Auto extraction — `src/research-engine/extraction/autoExtract.js` (539 lines) + `heuristicExtract.js` + `patternExtract.js` + `outcomeMatch.js` — EXTEND
`autoExtract({pages, abstract, protocol, baseStudy, at, idFn})` is a precision-minded PROSE pipeline: sentence-scoped clause-local outcome matching, positional ratio binding, parks unmatched stats, detectedOutcomes chooser feed, auto-computed ES via calcES with conversions[]. Disposition: new `surveyExtract.js` implements the §19 document survey (tables → prose fallback → figure flags → no-match chooser) reusing autoExtract's prose engine as its fallback stage; synonyms extended + versioned.

## 4. Shared contracts (agreed before parallel implementation, §8.9)

- **C1 Normalized item:** `{str, x, y, w, h}` (PDF user space, y↑, baseline origin) + optional `fontName`, `source: 'pdf-text'|'ocr'`, `itemIndex`. `normalizeItems` stays the single entry point; it is widened (additively) to carry `fontName`/`source`/`itemIndex` through when present.
- **C2 Grid contract (`buildTableGrid(items, opts) → TableGrid`):**
  `{ caption: string|null, captionEvidence, footnotes: string[], headerRows: number[], columns: [{x0, x1, align: 'left'|'right'|'decimal', headerPath: string[]}], rows: [{cells: [{raw, value: ParsedCell|null, bbox, colSpan?}], indentLevel, parentRow: number|null, bbox, kind: 'body'|'section'}], confidence: number, warnings: string[], parserVersion: string }`.
  Stages (each exported + unit-tested): `repairTokens` → `buildLines` → `stripCaptionAndFootnotes` → `inferColumns` → `assignCells` → `mergeWrappedRows` → `detectHeaderRows` → confidence. `PARSER_VERSION` exported constant, bumped on behavior change.
- **C3 ParsedCell (`cellGrammar.parseCell(raw)`):** discriminated union or null, never throws, raw always preserved: `INT{value}`, `FLOAT{value}`, `PCT{pct, count?}`, `N_OF_N{numerator, denominator}`, `MEAN_SD{mean, sd}`, `CI{low, high, warning?}` (reversed bounds preserved + warned, never reordered), `P{operator, value}`, `MISSING{marker}`.
- **C4 snapToken(str, clickOffset, selectionRange?)** → richest token containing offset (or covered by selection), nearest-token fallback within 3 chars; kinds from C3 plus composites (`ratioCI{est,lo,hi}`).
- **C5 Shape result (`detectTableShape(grid, pico)`):** `{shape: 'effect-per-row'|'arms-in-columns'|'two-by-two'|'mean-sd'|'single-arm'|'unknown', mapping: {columnTags, rowKind, armAssignment?}, confidence, evidence: string[], alternates: [{shape, confidence}]}` (alternate listed when top-two gap < 0.15).
- **C6 Provenance (additive on the existing `{method, page, region, excerpt, at}`):** table adds `{rowIndex, rowLabel, colIndexes, rawCells, shape, parserVersion, pdfFingerprint, sourceIdentity, tableLabel}`; figure adds `{figureType, calibration: {xFit, yFit, auto}, points, pdfFingerprint, sourceIdentity}`; click adds `{bbox, snippet, parsedToken, pdfFingerprint, sourceIdentity}`; prose adds `{sentenceRange, matchEvidence, parserVersion, pdfFingerprint, sourceIdentity}`. User corrections keep machine provenance + `{userCorrected: true, original: {raw, value}}`.
- **C7 sourceIdentity:** deterministic djb2-style hash string of `(pdfFingerprint | page | normalized region | rowIndex | colIndexes | method | outcomeId | timepoint | parserVersion)` — NOT random. `mkSourceIdentity(parts)` in `draftReconcile.js`.
- **C8 Precedence (`valuePrecedence.decideWrite({existingValue, existingOrigin, incoming})`):** → `{action: 'write'|'propose-replace'|'keep-existing'|'add-alternative', reason}`. Origin ladder: user-typed > user-corrected > user-confirmed draft > machine draft > empty. Machine never outranks human.
- **C9 Draft states:** `draft` (machine, needsReview) → `confirmed` | `corrected` (userCorrected flag) | `dismissed` (persisted in `project.extractionDismissed[]` by sourceIdentity — restorable) | `superseded` (source changed) | `sourceUnavailable`.
- **C10 AxisFit (serializable):** `{scale: 'linear'|'log', p0, p1, v0, v1, r2, ticks: [{px, value, raw}], domain: [lo, hi], warnings: string[]}` + reconstructed `toData`/`toPx`. Gates for auto-accept: r2 ≥ 0.999, ≥3 ticks, monotonic both axes, spacing sane, plausible bounds; any failure → confirm mode.
- **C11 OCR items:** normalized to C1 with `source:'ocr'` + per-word `confidence`; mapped to PDF user space via the render viewport transform; cache key `(pdfFingerprint, page, mode, tessVersion, scale)`.
